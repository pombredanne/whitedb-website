Release notes

whitedb-0.7.0.tar.gz

MD5 fingerprint: 1fc87b02eaf230fc45c040a2ba33bbd0

The 0.7.0 release is the latest one, featuring several improvements over 0.6.1:

  • The simplified query mechanism wg_find_record added.
  • wg_attach_existing_database added.
  • A fix for large memory segments on 32-bit architectures.
  • Error messages printed only to stderr.
  • An improved toolset and examples including dserve.

The 0.7.0 release contains a few non-documented features in active development and has not been through much testing yet.

We recommend to download 0.7.0.

whitedb-0.6.1.tar.gz

MD5 fingerprint: 37d9c25e973184db55f3e985d207f2da

The 0.6.1 release is stable. It has seen a lot of testing and use in production systems. Except for the remarks about 0.7.0 above, the functionality is basically the same as in 0.7.0.

Development on Github

The Github main branch is used for active development. You will find unfinished components and features.

There is no configure script on the main branch: use the provided bootstrap script running aclocal, autoheader, autoconf, libtoolize and automake to build the configure script.
